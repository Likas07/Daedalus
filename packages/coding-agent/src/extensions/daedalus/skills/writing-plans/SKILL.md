# Writing plans

Use this skill when turning a goal into an implementation plan.

- Clarify the outcome, constraints, and non-goals before planning.
- Break work into ordered, verifiable tasks with specific files or modules when known.
- Include dependencies, risks, rollback notes, and targeted verification commands.
- Keep the plan executable: each task should have a clear definition of done.
