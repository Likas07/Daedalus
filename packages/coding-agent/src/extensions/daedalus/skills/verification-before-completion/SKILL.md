# Verification before completion

Use this skill before claiming that work is done or tests pass.

- Run the relevant verification commands fresh after the final code change.
- Report exact commands, pass/fail status, and any skipped or blocked checks.
- If a check fails, distinguish pre-existing failures from regressions only with evidence.
- Do not describe work as complete unless the requested verification has passed or the blocker is explicit.
